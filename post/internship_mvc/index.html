MVC

MVC stands for Model View Controller

It is a design pattern that is used to separate the business logic from its display in a way that is easy to understand and implement. By separating the view and the logic a designer and a programmer can work hand in hand only communicating over a controller in the middle. You can think of the controller as an interface or an adapter used to connect the two worlds.

How it works is that upon receiving a request in the view/frontend, the frontend will send the information to the controller which then can send the information to the models that handle the logic and also send off information to the persistence layer.

More often than not an MVC pattern is used together with an observer pattern so that the controller can react to changes in the model and reflect them in the view.

Advantages: * Separation of view and logic * Isolated implementation of the business logic * Independent classes * Widely adopted pattern in many frameworks and project

Disadvantages: * Additional classes * Needed knowledge of MVC * View is strongly connected to the controller

The MVC design pattern is widely adopted, as I already mentioned. If you want to further separate the different parts of the system you may want to look into MVVM which is a pattern that is even less coupled than MVC.
